**1/**
doma puts .com domains onchain as tokens. normal part. weird part — they have
a markdown file at `/.well-known/skills/trade-tokens/SKILL.md` that tells an AI
agent how to buy and sell those domains. the file IS the integration. no sdk,
no docs portal, no api keys.

**2/**
domains are a 360 billion dollar market and buying a good one still works like
selling a used couch. DM the guy. argue. escrow.com. wait two weeks. pray he
doesn't ghost. i genuinely cannot believe this is how .com's trade in 2026.

**3/**
ok so tokenized domain, whatever. but the title and the money live on the same
chain now. no middleman. no 15% broker fee. no "i'm out of office until the
18th." you pay, it moves, done in one block.

**4/**
SKILL.md thing deserves more attention. it's literally just plain english
instructions for a bot. "to list a domain, sign this, post it here." anthropic
cooked up the format, doma is using it for an actual asset market. feels
obvious in hindsight.

**5/**
payments go through x402. you hit a url, server says 402 payment required,
agent sends USDC, retries, gets the thing. http 402 has been reserved and
unused since 1997. sat there for 29 years. turns out it was a machine-economy
feature the whole time.

**6/**
real example. you're starting something, need a name. "find a 2 word .com
under 10k, no trademark problems, no seo garbage in its past." agent goes,
reads listings, bids on a few, wins one. it shows up in your wallet. you never
opened a tab.

**7/**
fund angle — every big domain portfolio is just some guy with a spreadsheet
and 400 names. onchain those same 400 names can go into a vehicle, get
fractionalized, used as loan collateral, rented for a weekend campaign.
changes who's willing to touch the category.

**8/**
doma's one project but the shape matters. a skill file + a payment endpoint +
a token. that's probably what half the internet looks like once agents are the
main customer. worth checking what's in a site's /.well-known/ folder these
days. answer is getting interesting.
